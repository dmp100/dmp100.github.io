### 1단계: 개발자 도구로 실험하기 (1-2주)

먼저 브라우저의 개발자 도구를 최대한 활용하는 법을 익히세요. Chrome이나 Firefox의 개발자 도구를 열고(`F12` 또는 `Cmd+Option+I`) Elements 탭에서 CSS를 직접 수정해보면서 즉시 결과를 확인할 수 있습니다.

**실습 방법**: 현재 블로그를 열어두고, 개발자 도구에서 색상을 바꿔보거나, 여백을 조정하거나, 폰트를 변경해보세요. 마음에 드는 스타일을 찾으면 그때 CSS 파일에 적용하면 됩니다. 이렇게 하면 파일을 저장하고 새로고침하는 번거로움 없이 빠르게 실험할 수 있습니다.

**학습 포인트**: `display: flex`와 `display: grid`의 차이를 직접 바꿔보면서 레이아웃이 어떻게 변하는지 관찰하세요. `justify-content`와 `align-items`를 조정하면서 아이템들이 어떻게 정렬되는지 이해하는 것이 중요합니다.

### 2단계: Jekyll 구조 완전히 이해하기 (1주)

Jekyll은 정적 사이트 생성기로, 빌드 시점에 모든 페이지를 미리 생성합니다. 이 구조를 이해하면 훨씬 수월하게 작업할 수 있습니다.

**핵심 개념**: Jekyll은 Liquid 템플릿 언어를 사용합니다. `{% %}` 안의 코드는 로직을 처리하고, `{{ }}` 안의 코드는 변수를 출력합니다. 예를 들어 `{{ page.title }}`은 현재 페이지의 제목을 출력하고, `{% if page.status == "active" %}`는 조건문을 실행합니다.

**실습 방법**: `_layouts` 폴더의 파일들을 하나씩 열어보고, 어떤 변수들이 사용되는지 확인하세요. 그리고 해당 변수가 어디서 정의되는지(`_config.yml`인지, 각 페이지의 front matter인지) 추적해보세요.

**디버깅 팁**: Jekyll 빌드 중에 에러가 나면 터미널에 정확한 줄 번호가 표시됩니다. 해당 파일의 그 줄을 확인하고, Liquid 문법이 올바른지 체크하세요. 자주 발생하는 실수는 `{% endif %}`나 `{% endfor %}` 같은 닫는 태그를 빠뜨리는 것입니다.

### 3단계: CSS Flexbox와 Grid 마스터하기 (2주)

현대적인 레이아웃은 대부분 Flexbox와 Grid로 만들어집니다. Android의 ConstraintLayout을 다뤄본 경험이 있다면 개념적으로 비슷한 부분이 많습니다.

**Flexbox 학습**: Flexbox는 1차원 레이아웃(가로 또는 세로 한 방향)에 적합합니다. 부모 요소에 `display: flex`를 설정하고, 자식 요소들이 어떻게 배치될지를 `flex-direction`, `justify-content`, `align-items` 등으로 제어합니다.

**Grid 학습**: Grid는 2차원 레이아웃(가로와 세로 동시)에 적합합니다. `grid-template-columns`로 열의 크기를 정의하고, `gap`으로 간격을 설정합니다. 예를 들어 `grid-template-columns: 150px 1fr`은 첫 번째 열은 150px 고정, 두 번째 열은 나머지 공간 전체를 차지합니다.

**추천 학습 리소스**: 
- Flexbox Froggy (https://flexboxfroggy.com/) - 게임으로 Flexbox 배우기
- Grid Garden (https://cssgridgarden.com/) - 게임으로 Grid 배우기
- MDN Web Docs - 정확한 레퍼런스 문서

**실습 프로젝트**: 프로젝트 상세 페이지의 레이아웃을 Flexbox에서 Grid로 바꿔보거나, 반대로 해보세요. 같은 결과를 다른 방법으로 만들어보면서 각각의 장단점을 이해할 수 있습니다.

### 4단계: 반응형 디자인 패턴 익히기 (1-2주)

모바일에서도 잘 보이는 사이트를 만드는 것은 필수입니다. 반응형 디자인의 핵심은 `@media` 쿼리입니다.

**기본 접근법**: 모바일 퍼스트로 작업하는 것을 추천합니다. 즉, 기본 스타일은 모바일용으로 작성하고, `@media (min-width: 768px)`처럼 화면이 커질 때 추가 스타일을 적용하는 방식입니다.

**브레이크포인트 선택**: 일반적으로 사용되는 브레이크포인트는 다음과 같습니다:
- 모바일: ~767px
- 태블릿: 768px~1023px  
- 데스크톱: 1024px~

**테스트 방법**: Chrome 개발자 도구의 Device Mode(`Cmd+Shift+M`)를 사용하면 다양한 화면 크기에서 어떻게 보이는지 즉시 확인할 수 있습니다.

### 5단계: JavaScript 기초 다지기 (2-3주)

정적 사이트라도 인터랙티브한 요소를 추가하려면 JavaScript가 필요합니다.

**시작하기 좋은 기능들**:
- 다크모드 토글 버튼 만들기
- 이미지 클릭 시 확대 보기 모달
- 스크롤 시 헤더 고정/숨김
- 페이지 로딩 애니메이션

**학습 순서**: 먼저 DOM 조작 기본(요소 선택, 클래스 추가/제거)을 익힌 후, 이벤트 리스너(클릭, 스크롤 등)를 다루고, 마지막으로 localStorage를 활용한 상태 저장을 배우세요.

**주의사항**: Jekyll은 빌드 타임에 페이지를 생성하므로, 사용자 상호작용이 필요한 부분은 반드시 JavaScript로 처리해야 합니다. Liquid 템플릿은 빌드 시에만 실행되고, 브라우저에서는 실행되지 않습니다.

### 6단계: 버전 관리와 배포 자동화 (1주)

GitHub Pages를 사용한다면, GitHub Actions를 활용해서 자동 배포를 설정할 수 있습니다.

**기본 워크플로우**: 코드를 `main` 브랜치에 푸시하면 자동으로 Jekyll 빌드가 실행되고, 생성된 사이트가 배포됩니다. 이 과정을 `.github/workflows/jekyll.yml` 파일로 정의할 수 있습니다.

**브랜치 전략**: `main` 브랜치는 항상 배포 가능한 상태로 유지하고, 새로운 기능은 별도 브랜치(`feature/레트로-스타일` 같은 이름)에서 작업한 후 머지하는 것이 안전합니다.

## 2단계: 개발자 도구로 먼저 실험하기


브라우저에서 `http://127.0.0.1:4000/projects/`를 열고, `F12` 키를 눌러 개발자 도구를 엽니다. 상단의 Elements 탭을 클릭하면 HTML 구조가 보이고, 오른쪽에는 해당 요소에 적용된 CSS 스타일이 보입니다.

좌측 상단의 선택 도구(화살표 아이콘)를 클릭한 후, 페이지에서 프로젝트 이름 중 하나를 클릭하세요. 그러면 Elements 탭에서 해당 요소가 자동으로 선택되고, 오른쪽 Styles 패널에서 `.exa-name` 클래스에 적용된 스타일들을 볼 수 있습니다.

이제 Styles 패널에서 `.exa-name` 섹션을 찾아서, 그 아래 빈 공간을 클릭해보세요. 여기에 새로운 CSS 속성을 직접 입력할 수 있습니다. 먼저 부드러운 전환 효과를 추가하기 위해 다음을 입력해보세요.

```css
transition: all 0.3s ease;
```

이 속성은 모든 CSS 변화가 0.3초에 걸쳐 ease 타이밍으로 일어나도록 만듭니다. 아직 눈에 보이는 변화는 없습니다. 왜냐하면 변화시킬 대상을 아직 정의하지 않았기 때문입니다.

이제 호버 상태를 정의해야 합니다. Styles 패널 상단의 `:hov` 버튼을 클릭하면 가상 클래스를 강제로 활성화할 수 있는 체크박스들이 나타납니다. `:hover`를 체크해보세요. 그러면 마우스를 올린 것처럼 상태가 변합니다.

하지만 아직 우리는 `.exa-name:hover` 스타일을 정의하지 않았으므로 변화가 없습니다. Styles 패널에서 `.exa-name:hover` 섹션을 찾거나, 없다면 직접 추가해야 합니다. 패널 상단의 `+` 버튼을 클릭하면 새로운 CSS 규칙을 추가할 수 있습니다. 선택자를 `.exa-name:hover`로 수정하고 다음 속성들을 추가해보세요.

```css
transform: scale(1.05);
color: #4fc3f7;
```

`:hover` 체크박스를 켜고 끄면서 효과가 부드럽게 적용되는지 확인해보세요. 마음에 든다면 이제 이 스타일을 실제 CSS 파일에 추가할 차례입니다.

## 3단계: 어떤 파일을 수정해야 하는지 찾기

All Projects 페이지의 스타일은 어디에 정의되어 있을까요? 이를 찾는 방법은 여러 가지가 있습니다. 가장 확실한 방법은 페이지의 HTML 소스를 직접 확인하는 것입니다.

브라우저에서 페이지를 우클릭하고 "페이지 소스 보기"를 선택하세요. HTML 코드가 표시되는데, 상단 부분에서 `<link rel="stylesheet">` 태그를 찾아보세요. 우리 경우에는 다음과 같은 줄을 찾을 수 있을 것입니다.

```html
<link rel="stylesheet" href="/assets/css/all-projects.css">
```

이것이 바로 우리가 수정해야 할 파일입니다. 경로를 보면 `assets` 폴더 안의 `css` 폴더에 있는 `all-projects.css` 파일이라는 것을 알 수 있습니다.

또 다른 방법은 개발자 도구의 Styles 패널에서 직접 확인하는 것입니다. 각 CSS 규칙 오른쪽에는 파일 이름과 줄 번호가 표시됩니다. `.exa-name` 규칙 옆을 보면 `all-projects.css:67` 같은 형태로 표시되어 있을 것입니다. 이는 `all-projects.css` 파일의 67번째 줄에 이 스타일이 정의되어 있다는 의미입니다.

## 4단계: CSS 파일 열고 수정하기

VSCode나 선호하는 에디터로 블로그 프로젝트를 열고, 파일 탐색기에서 `assets/css/all-projects.css` 파일을 찾아 엽니다. 파일 구조는 다음과 같습니다.

```
dmp100.github.io/
├── _layouts/
├── _posts/
├── assets/
│   └── css/
│       ├── all-projects.css  ← 이 파일을 수정할 것입니다
│       └── project.css
├── projects/
└── _config.yml
```

`all-projects.css` 파일을 열면 여러 CSS 규칙들이 보일 것입니다. `.exa-name` 클래스를 찾아보세요. 파일이 길다면 `Cmd+F` (Windows는 `Ctrl+F`)로 검색 기능을 사용하면 빠르게 찾을 수 있습니다.

`.exa-name` 규칙을 찾았다면, 그 안에 우리가 개발자 도구에서 실험했던 `transition` 속성을 추가합니다. 그리고 바로 그 아래에 `.exa-name:hover` 규칙을 새로 만듭니다. 코드는 다음과 같은 형태가 될 것입니다.

```css
.exa-name {
  text-decoration: none;
  font-weight: bold;
  margin-left: 15px;
  /* 부드러운 전환 효과를 위한 속성 추가 */
  transition: all 0.3s ease;
}

/* 마우스를 올렸을 때의 스타일 */
.exa-name:hover {
  /* 1.05배로 살짝 확대 */
  transform: scale(1.05);
  /* 색상을 더 밝게 변경 */
  color: #4fc3f7;
}
```

주석을 추가한 이유는 나중에 이 코드를 다시 볼 때 왜 이렇게 작성했는지 기억하기 위함입니다. 특히 `transition`의 `0.3s`나 `scale(1.05)`처럼 특정 숫자를 사용한 경우, 그 의도를 주석으로 남겨두면 나중에 값을 조정할 때 도움이 됩니다.

## 5단계: 결과 확인하기

파일을 저장하고 브라우저로 돌아가서 새로고침하세요. Jekyll은 파일 변경을 자동으로 감지하고 재빌드하지만, CSS 같은 정적 파일의 경우 브라우저 캐시 때문에 변경사항이 즉시 반영되지 않을 수 있습니다. 이런 경우 하드 새로고침을 해야 합니다.

Mac에서는 `Cmd+Shift+R`, Windows에서는 `Ctrl+Shift+R`을 누르면 캐시를 무시하고 새로고침됩니다. 이제 프로젝트 이름 위에 마우스를 올려보세요. 부드럽게 커지면서 색이 변하는 효과가 보일 것입니다.

만약 효과가 보이지 않는다면, 다음 사항들을 확인해보세요. 첫째, CSS 파일을 저장했는지 확인하세요. 둘째, Jekyll 서버의 터미널 출력을 확인해서 에러가 없는지 보세요. 셋째, 브라우저의 개발자 도구 Console 탭에서 에러 메시지가 있는지 확인하세요.

## 6단계: 찾아서 공부하는 방법

작업 중에 모르는 것이 생기면 어떻게 찾아서 공부할까요? 단계별로 접근하는 것이 중요합니다.

### 공식 문서부터 시작하기

CSS 속성을 검색할 때는 항상 "MDN"을 함께 검색하세요. 예를 들어 `transform` 속성에 대해 알고 싶다면 "CSS transform MDN"으로 검색하면 됩니다. MDN은 Mozilla Developer Network의 약자로, 웹 표준에 대한 가장 정확하고 신뢰할 수 있는 문서를 제공합니다.

MDN 문서는 일반적으로 다음과 같은 구조로 되어 있습니다. 상단에는 속성에 대한 간단한 설명이 있고, 그 아래에는 문법(Syntax) 섹션에서 어떻게 사용하는지 보여줍니다. 더 아래로 스크롤하면 예제(Examples) 섹션에서 실제 사용 사례를 볼 수 있고, 맨 아래에는 브라우저 호환성(Browser Compatibility) 정보가 있어서 어떤 브라우저에서 지원되는지 확인할 수 있습니다.

### 에러 메시지 읽는 법

에러가 발생하면 당황하지 말고 에러 메시지를 천천히 읽어보세요. 에러 메시지는 대부분 무엇이 잘못되었고, 어디서 문제가 발생했는지 알려줍니다.

예를 들어 Jekyll 빌드 중에 다음과 같은 에러가 발생했다고 가정해봅시다.

```
Liquid Exception: Liquid syntax error (line 15): Unknown tag 'endfi' in _layouts/all-projects.html
```

이 메시지를 분해해서 읽어보면, Liquid 문법 에러가 발생했고, `_layouts/all-projects.html` 파일의 15번째 줄에 있으며, `endfi`라는 알 수 없는 태그가 문제라는 것을 알 수 있습니다. 아마도 `{% endif %}`를 잘못 타이핑해서 `{% endfi %}`라고 쓴 것일 가능성이 높습니다.

### 비슷한 예제 찾아보기

완전히 새로운 기능을 추가할 때는 비슷한 예제를 찾아보는 것이 도움이 됩니다. "CSS hover effect example"처럼 검색하면 CodePen이나 CSS-Tricks 같은 사이트에서 다양한 예제를 찾을 수 있습니다.

하지만 예제 코드를 그대로 복사-붙여넣기하는 것은 좋지 않습니다. 대신 예제를 읽어보면서 어떤 원리로 작동하는지 이해하려고 노력하세요. 각 줄이 무엇을 하는지 주석으로 메모를 달아보면 이해하는 데 도움이 됩니다.

### 개발자 도구로 다른 사이트 분석하기

마음에 드는 효과를 다른 웹사이트에서 발견했다면, 그 사이트의 개발자 도구를 열어서 어떻게 만들었는지 직접 확인할 수 있습니다. 해당 요소를 선택하고 Styles 패널에서 어떤 CSS 속성들이 사용되었는지 보세요. 특히 `:hover` 상태에서 어떤 변화가 일어나는지 주의 깊게 관찰하세요.

## 7단계: 변경사항 커밋하기

작업이 완료되고 결과가 만족스러우면, Git으로 변경사항을 커밋하세요. 커밋 메시지는 무엇을 변경했는지 명확하게 작성하는 것이 중요합니다.

```bash
git add assets/css/all-projects.css
git commit -m "feat: 프로젝트 이름에 호버 효과 추가

- 마우스 오버 시 1.05배 확대 효과
- 색상 변경 효과 (#4fc3f7)
- 0.3초 부드러운 전환 애니메이션"
```

커밋 메시지의 첫 줄은 50자 이내로 간단히 작성하고, 빈 줄을 하나 띄운 후 더 자세한 설명을 추가합니다. `feat:`는 새로운 기능을 추가했다는 의미의 접두사입니다. 버그를 고쳤다면 `fix:`, 문서를 수정했다면 `docs:`, 코드 정리만 했다면 `chore:` 같은 접두사를 사용합니다.

## 8단계: 다음 단계로 나아가기

기본적인 호버 효과가 완성되었다면, 이제 조금씩 개선해나갈 수 있습니다. 예를 들어 다음과 같은 추가 효과들을 시도해볼 수 있습니다.

커서 모양을 변경하고 싶다면 `cursor: pointer;`를 추가할 수 있습니다. 이렇게 하면 마우스 오버 시 손가락 모양으로 바뀌어 클릭 가능하다는 것을 명확히 알려줍니다.

텍스트에 그림자 효과를 추가하고 싶다면 `text-shadow: 0 0 10px rgba(79, 195, 247, 0.5);`를 사용할 수 있습니다. 이는 텍스트 주변에 부드러운 빛 효과를 만들어줍니다.

애니메이션의 타이밍 함수를 바꾸고 싶다면 `transition`의 `ease` 부분을 `ease-in-out`, `ease-in`, `cubic-bezier(0.68, -0.55, 0.265, 1.55)` 같은 다른 값으로 변경해볼 수 있습니다. 각각 다른 느낌의 애니메이션을 만들어냅니다.

## 실전 팁: 문제가 생겼을 때

작업 중에 예상치 못한 문제가 생기는 것은 아주 자연스러운 일입니다. 중요한 것은 체계적으로 문제를 해결하는 방법을 아는 것입니다.

첫째, 가장 최근에 변경한 것부터 되돌려보세요. 만약 새로운 CSS 규칙을 추가한 후 레이아웃이 망가졌다면, 그 규칙을 주석 처리해보세요. CSS에서 주석은 `/* */`로 감싸면 됩니다. 문제가 해결되면 해당 규칙에 문제가 있다는 것을 알 수 있습니다.

둘째, 개발자 도구의 Computed 탭을 활용하세요. Elements 탭에서 요소를 선택한 후 Computed 탭을 보면, 최종적으로 적용된 모든 스타일 값을 볼 수 있습니다. 예상한 값과 다르다면 어떤 규칙이 이를 덮어썼는지 확인할 수 있습니다.

셋째, 특정성(Specificity) 문제일 수 있습니다. CSS는 더 구체적인 선택자가 우선순위를 갖습니다. 예를 들어 `.exa-line .exa-name`은 단순히 `.exa-name`보다 더 구체적입니다. 만약 스타일이 적용되지 않는다면, 선택자를 더 구체적으로 만들거나, 최후의 수단으로 `!important`를 사용할 수 있습니다. 하지만 `!important`는 가능한 피하는 것이 좋습니다.

이렇게 하나씩 문제를 해결해나가다 보면, 점점 더 빠르고 자신 있게 작업할 수 있게 됩니다. 처음에는 작은 변경 하나에도 시간이 오래 걸리지만, 반복하다 보면 자연스럽게 빨라집니다. 중요한 것은 포기하지 않고 계속 시도하는 것입니다.
